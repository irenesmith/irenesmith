Modernizing a website

Here is a discussion of what you need to change if you are "lucky" enough to have to update an old website. Some of the things you must do because the existing features are deprecated or removed altogether. Other's are just best practices that probably didn't exist ten or fifteen years ago.

Steps to follow

Step 1: Remove deprecated attributes and tags

Start by removing deprecated attributes and tags. Define fonts, width and height, margins, and borders using CSS instead of attributes or tags. This has the huge advantage that if you change your mind, you can change it anywhere with a single edit.

This means that:

<p align="center"><font face="Times New Roman">
  This is a paragraph
</font></p>

will become:

<p class="centered">
  This is a paragraph
</p>

with this CSS:

body {
  font-family: "Times New Roman", "serif";
}

p.centered {
  text-align:center;
}

This looks like more code but, if you had six paragraphs and each of them had the same HTML, you're repeating that code over and over while the modernized page has the definition for fonts and alignment only once. And if, six months from now, you decide you would rather use Arial and have the paragraphs right-aligned, you would have to make the change in all six paragraphs. Now imagine having six to ten paragraphs on fifty different web pages. What are the chances that you will forget to change at least one?

The img tag now requires an alt attribute. The contents of the attribute should be a description of the image. However, if the image is purely decorative, leaving it empty alt="" descriptive software won't bother mentioning it. Sor, for example, it means nothing to a vision-impaired visitor to know that you have a separator bar on the page. That's why all of the example bar images have an alt tag set to the empty string.

Step 2: Consider your fonts and color palette

Early web pages sometimes used some bizarre color combinations. Not only text on a solid colored background, but text on super-busy patterns.

Step 3: Replace table layout

Unless you are actually displaying tables of data, replace tables with an alternative layout. Use divs and CSS grid or CSS flexbox layout instead. Tables slow down page load because the browser has to render the entire table. Also, tables quickly get out of hand and you end up with multiple nested tables. If nothing else, it makes your marked up source code more difficult to read.
