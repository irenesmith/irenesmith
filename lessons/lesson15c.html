For example, this page uses several tables to get things where they belong. In order to show you how I've made this page look the way it does, I've also created a second page for this layout that has the borders of the large table turned on so you can see where the breaks are.

I actually had a rather difficult time with this part of the lesson because I started out by creating a rather elaborate page that looked fine in Internet Explorer but terrible in Netscape and Firefox. For some reason, despite the fact that I defined a table that was centered on the page, both Netscape and Firefox absolutely refused to do anything with it but stick it right up against the left border.

I tried everything I could think of to solve the problem and finally started from scratch. Strangely enough, this page is pretty much the same as the original. The only difference is that, instead of defining CSS styles to control the size of the table cells, I've done it explicitly in the HTML markup instead. Also, instead of creating an XHTML page as I've been doing, I created a simple HTML page. Is that what makes the difference? I'm not sure, but I'm glad I got it to work.
